Foire Aux Questions¶

Problèmes courants¶

Si CryptPad ne fonctionne pas comme attendu, il existe quelques solutions courantes à essayer. La plupart d'entre elles concernent les navigateurs web et les extensions qui interfèrent avec les fonctions de sécurité de CryptPad ou désactivent les fonctionnalités de base du navigateur.

  1. Vérifiez la liste des extensions ci-dessous et désactivez celles que vous avez installées, le cas échéant

  2. Essayez d'utiliser votre navigateur dans une fenêtre privée et/ou en désactivant toutes les extensions

  3. Essayez d'utiliser un autre navigateur. Nous recommandons les dernières versions de Firefox ou Chromium

  4. Si les problèmes persistent, veuillez contacter le Support de votre instance

Liste des extensions connues pour causer des problèmes avec CryptPad¶

  • Librejs

  • Adblock

  • Night Eye

  • uMatrix

  • JShelter

  • AdGuard

  • SuperAgent

  • Tampermonkey

  • Arkenfox

  • Loom

  • MozBar

  • Adblocker Stands

  • Emoji keyboard emojis for chrome

  • PAYBACK Internet Assistent (PIA)

  • User-Agent Switcher and Manager

Note

Si vous identifiez des problèmes avec une autre extension, veuillez le reporter à l'équipe de développement

J'ai oublié mon mot de passe¶

Les administrateur·ices d'une instance CryptPad n'ont pas la possibilité de restaurer ou de réinitialiser les mots de passe des utilisateur·ices. Cela garantit également qu'ils ne peuvent pas accéder à leurs données (par exemple, le contenu des documents).

Si vous êtes toujours connecté à un compte dont vous avez oublié le mot de passe, vous pouvez transférer tous les documents vers un nouveau compte en suivant les étapes suivantes :

  1. En étant connecté au compte avec le mot de passe perdu, naviguez vers : Menu utilisateur·ice (avatar en haut à droite) > Préférences > CryptDrive

  2. Cliquez sur le bouton Sauvegarder pour télécharger un fichier contenant les clés de tous les documents. (note : ceci est différent de Télécharger mon Cryptdrive qui sauvegarde le contenu de tous les documents individuels)

  3. Dans un autre navigateur, créez un nouveau compte avec un mot de passe connu

  4. En étant connecté au nouveau compte : Menu utilisateur·ice (avatar en haut à droite) > Préférences > CryptDrive et cliquez sur le bouton Restaurer

Comment importer/exporter mes documents vers/depuis un autre logiciel ?¶

Rendre CryptPad compatible avec une large gamme de formats de documents est un enjeu de taille pour l'équipe de développement. Pendant que le travail dans ce domaine se poursuit, les méthodes ci-dessous permettent d'importer et d'exporter depuis/vers d'autres logiciels largement utilisés.

Note

Un simple copier-coller du contenu depuis un autre logiciel vers CryptPad est une première étape facile à essayer pour l'import/export. Veuillez noter que les méthodes ci-dessous peuvent cependant améliorer considérablement les résultats.

Il est possible d'importer des fichiers sur CryptPad pour un stockage et un partage sécurisés, mais cela ne fait pas de chaque fichier un document modifiable par les applications CryptPad. Pour ce faire, vous avez besoin de :

  • Click droit sur le fichier téléversé > Ouvrir dans l'éditeur, ou...

  • Créez un nouveau document vide et utilisez la fonctionnalité Fichier > Importer dans l'application CryptPad appropriée, comme expliqué ci-dessous.

Avertissement

  • Le navigateur web Safari , utilisé dans macOS, iPadOS et iOS, n'est pas supporté pour le moment

  • Le support du format de fichier CSV est actuellement cassé, les imports/exports .csv sont désactivés pour le moment

  • Le support pour les formats OpenDocument, les imports/exports .odt, .odp et .ods sont actuellement expérimentaux et sujets à erreurs

Si vous rencontrez des problèmes, veuillez jeter un oeil aux Problèmes courants en premier. Si vous n'arrivez pas à résoudre votre problème en suivant ces instructions, veuillez contacter vos administrateur·ices d'instance en utilisant le service d'assistance Support. Fournissant un document illustrant votre problème avec la fonctionnalité Partager un lien.

Import/export n'est pas visible depuis le menu Fichier¶

Pour les tableurs et les fonctionnalités d'éditions de documents avancées, CryptPad intègre OnlyOffice. Il en résulte deux barres d'outils, chacunes avec leur propre menu Fichier au sein du même écran. Veuillez vous assurer de vérifier dans chaque menu lorsque vous cherchez l'import/export.

Import/export de l'application Document¶

L'application Document support les formats Microsoft Word .docx et OpenDocument .odt. Ces deux formats peuvent être exportés à partir de logiciels courants tels que Google Docs et LibreOffice Writer :

  1. Exporter le document au format .docx ou .odt depuis votre logiciel existant

  2. Enregistrez le fichier sur votre ordinateur

  3. Créez un nouveau fichier Document sur CryptPad (ou un document existant pour remplacer le contenu)

  4. Fichier > Importer et sélectionnez votre fichier

Pour exporter, utiliser Fichier > Exporter et sélectionner un format :

  • Pour utilisation dans Microsoft Word ou Google Docs, sélectionner .docx

  • Pour utilisation dans LibreOffice Writer, sélectionner .odt

  • Pour les cas où aucune autre modification des données n'est nécessaire après l'exportation (lettres, rapports, etc.), sélectionner .pdf

  • Pour faire une sauvegarde ou pour partager avec un·e utilisateur·ice sur une autre instance CryptPad, sélectionner .bin

Import/export de l'application Présentation¶

L'application Présentation supporte les formats Microsoft PowerPoint .pptx et OpenDocument .odp. Ces deux formats peuvent être exportés à partir de logiciels courants tels que Google Slides et LibreOffice Impress :

  1. Exporter votre document au format .pptx ou .odp depuis votre logiciel existant

  2. Enregistrez le fichier sur votre ordinateur

  3. Créez une nouvelle présentation sur CryptPad (ou un document existant pour remplacer le contenu)

  4. Fichier > Importer et sélectionnez votre fichier

Pour exporter, utiliser Fichier > Exporter et sélectionner un format :

  • Pour utilisation dans Microsoft PowerPoint ou Google Slides, sélectionner .pptx

  • Pour utilisation dans LibreOffice Impress, sélectionner .odp

  • Pour les cas où aucune autre modification des données n'est nécessaire après l'exportation (présentations, rapports, etc.), sélectionner .pdf

  • Pour faire une sauvegarde ou pour partager avec un·e utilisateur·ice sur une autre instance CryptPad, sélectionner .bin

Import/export de l'application Tableur¶

Pour exporter, utiliser Fichier > Exporter et sélectionner un format :

  • Pour utilisation dans Microsoft Excel ou Google Sheets, sélectionner .xlsx

  • Pour faire une sauvegarde ou pour partager avec un·e utilisateur·ice sur une autre instance CryptPad, sélectionner .bin

Import/export de l'application Documents texte¶

L'application Documents texte supporte le format HTML pour l'import/export :

  1. Exporter le document au format HTML depuis votre logiciel existant

  2. Enregistrez le fichier sur votre ordinateur

  3. Créez un nouveau Documents texte sur CryptPad (ou un document existant pour remplacer le contenu)

  4. Fichier > Importer et sélectionnez votre fichier

Pour exporter, utiliser Fichier > Exporter et sélectionner .doc comme format.

Je ne peux pas faire X avec un document que j'ai créé¶

Certaines fonctionnalités sur CryptPad sont réservées aux propriétaires du document. Cela inclut :

  • Activer une liste d'accès

  • Ajouter un mot de passe

  • Ajouter ou supprimer d'autres propriétaires

  • Détruire le document

La propriété d'un document est établie lors de sa création et ne peut pas être modifiée par la suite si le document est créé sans propriétaire.

L'équipe de développement envisage de supprimer cette distinction à l'avenir. Pour le moment, l'option la plus sûre est d'être "propriétaire" de tous les documents que vous créez afin de conserver le contrôle sur ceux-ci, y compris la possibilité de les détruire.

Puis-je utiliser CryptPad sur mobile ?¶

CryptPad est conçu pour fonctionner le mieux possible sur les petits écrans. En fonction des performances de votre appareil, il devrait être possible d'utiliser CryptPad sur un téléphone portable. Des travaux visant à améliorer cet usage ont été entrepris en 2020. Si vous remarquez des points à améliorer à cet égard, veuillez contacter Support ou le signaler sur Github.

Avez vous l'intention de faire une application ?¶

Nous ne prévoyons pas d'application mobile pour les raisons suivantes :

  • Cela augmenterait considérablement la quantité de code à développer et à maintenir, créant de fait plusieurs "versions" de CryptPad pour iOS et Android

  • CryptPad est un logiciel libre et peut être hébergé par quiconque souhaite proposer ce service. Par conséquent, les utilisateur·ices d'une application mobile devraient spécifier à quelle instance CryptPad ils veulent se connecter, ce qui serait déroutant. Pour compliquer encore les choses, chaque instance peut exécuter une version différente du logiciel, suivant la dernière mise à jour appliquée par les administrateur·ices

Pour résoudre ces problèmes, l'équipe de développement travaille à faire du CryptPad une "application web progressive". Cela signifie que le logiciel peut être utilisé sur un mobile par le biais du navigateur, se comportant comme une application tout en étant le même que celui qui fonctionne sur les navigateurs de bureau. Cela présente l'avantage de transformer chaque instance de CryptPad en un fournisseur d'application web, plutôt que de faire peser sur l'utilisateur la tâche de choisir la bonne instance.

CryptPad peut-il synchroniser des documents avec mon ordinateur ?¶

La façon dont le chiffrement est actuellement utilisé dans le CryptPad ne permet pas la synchronisation avec le système de fichiers local. Cependant, cette fonctionnalité est régulièrement demandée, et l'équipe de développement espère la rendre disponible à l'avenir.

Quelle est la relation entre CryptPad et OnlyOffice ?¶

Les applications CryptPad Document, Présentation et Tableur sont une intégration d'OnlyOffice Docs. Cependant, cela ne concerne que le code côté client, le CryptPad n'utilise pas le serveur de documents OnlyOffice. La collaboration chiffrée du CryptPad, utilisée pour les feuilles de calcul et autres applications, est complètement différente du système de chiffrement utilisé dans certaines parties du OnlyOffice en amont. Certains des outils de conversion de format de fichier du CryptPad sont basés sur le code OnlyOffice, mais un travail important a été effectué pour le faire fonctionner dans le navigateur plutôt que sur le serveur, évitant ainsi de devoir révéler le contenu des documents des utilisateurs lors de la conversion.

À quel point CryptPad est sécurisé ?¶

CryptPad vise à protéger l'identité de ses utilisateur·ices et leur contenu de nous et d'acteurs malveillants externes. Cependant, cette sécurité ne peut être parfaite et nécessite de bonnes pratiques suivies par les utilisateur·ices ainsi de faire confiance à l'instance que vous utilisez. Plus d'explications et recommandations sont disponibles sur notre blog.

Pour résumer les points principaux :

  • Les administrateur·ices de l'instance sont sont considéré·es comme honnêtes mais curieux·ses, ce qui veut dire que même si iels suivent les règles, iels peuvent essayer d'obtenir un maximum d'information grâce à ce qu'iels reçoivent. Nous maintenons une liste d'instances CryptPad publiques pour lesquelles nous avons vérifié qu'elles sont à jour et proprement configurées;

  • Nous ne pouvons nous fier aux canaux de communications qui peuvent être activement malveillants : des menaces externes sont à même d'altérer, relayer ou supprimer des messages. Ceci n'impacte en rien la sécurité de CryptPad;

  • Les utilisateur·ices avec qui vous partagez votre document sont honnêtes à partir du moment où iels reçoivent vos données, iels sont responsables de ne pas les communiquer à des tiers. Cependant, votre identité reste incertaines pour elleux, dans une forme faible d'anonymat : iels ont accès à votre clé publique et votre nom d'affichage ;

    • Ainsi, le lien de partage doit être considéré aussi sensible qu'un mot de passe. Pour ajouter un niveau de sécurité supplémentaire, nous recommandons d'ajouter un mot de passe pour accéder à vos fichiers. Si vous avez un compte, il est stocké dans votre drive (qui est considéré comme sécurisé) et l'accès reste transparent. Cependant, des utilisateur·ices externes trouvant le lien de partage de votre document auront besoin du mot de passe pour y accéder.

  • L'historique entier du documents et les différent·es collaborateur·ices sont visibles quand vous partagez un document. Si vous souhaitez garder ces information secrètes, la meilleure manière (à ce jour) est de partager une copie du document, qui lorsqu'elle est créée commence avec un nouvel historique.

FAQ pour cryptpad.fr¶

Puis-je payer mon abonnement sans utiliser de carte ?¶

Nous acceptons les paiements par virement bancaire, Paypal, ou autre selon les demandes. Cependant, en raison du travail supplémentaire qu'implique l'émission de factures et le traitement manuel des paiements, cette option est limitée aux plans annuels pour nos offres Duo et supérieures et entraîne une surcharge de 30%.

Acceptez-vous les crypto-monnaies pour les abonnements et les dons ?¶

Nous acceptons les paiements en Bitcoin. Ceux-ci sont soumis aux suppléments de traitement manuel comme expliqué dans la question précédente. En outre, en tant que société enregistrée en France, nous sommes légalement tenus de vous demander un nom et une adresse pour l'émission de la facture, ainsi qu'une adresse électronique pour la communication concernant votre abonnement.

Pouvez-vous fournir un accord de traitement des données (DPA) ?¶

Les DPA sont une option de nos Plans organisation, notez qu'un exemple de DPA est disponible en aperçu sur cette page.
